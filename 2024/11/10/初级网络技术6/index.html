什么是路由？

路由：把用户数据从一个网络发送到另一个网络

路由器核心目的：数据转发，选择最佳路径转发

怎么选择出的最优路径呢？？基于路由表选出最佳路径，然后再进行数据的转发

什么是路由表？路由器的最核心的一个功能

路由表：

路由信息存储在路由表中

路由表是路由器转发数据的依据

查看路由表：dis ip routing-table

一些相关的协议：

direct 直接连接的

static 静态 手动向路由表添加目的网段

动态路由协议：

rip用于小型网络）

ospf

优先级（管理距离）：

越小越优，static 60，rip100，direct 0 ospf 10 华为体系

思科体系：direct 0，static 1，ospf 110， rip 120

cost：

也叫做开销值，经过路由器的个数，谁经过越少，开销值小

不同协议比较优先级，相同协议比较开销值，不同协议比较开销值没意义

下一跳：

给流量走向指向方向

interface：

Destination

目的网段

路由优选：

• 到达同一个目标网络可能有多个路由源、多条路径

网络号相同、子网掩码相同

只有最佳路由才会进入路由表

• 路由选路原则

• 管理距离越小越优先

• 度量值越小越优先

不同路由协议的度量值不具备可比性

路由表中如果有多个匹配目的网络的路由条目，则路由器会选择

掩码最长的条目。

最长延码匹配规则，路由表中如果有多个匹配目的网络的路由条目，则路由器会选择掩码最长的条目，因为掩码长，网段里面的ip少，容易找到

路由器的功能：

路由：建立并维持路由表，正确描述网络拓扑

数据转发：选择最佳路径把ip包封装并从出口转发出去；为什么要封装？、因为核对ip的时候拆过包了

控制平面：建立和维护路由表

数据平面：查找路由表，转发数据

功能收敛：路由器对网络拓扑结构有了一致的看法，通过ospf rip 这些学习后就知道了网络的结构

什么是静态路由：

由是指由管理员手动配置和维护的路由。

静态路由配置简单，被广泛应用于网络中。另外，静态路由还可以实现负载

均衡和路由备份。因此，学习并掌握好静态路由的应用与配置是非常必要的。

静态路由具体配置

实验目的：

将r1，r2，r3，r4，配置如图所示的网段ip，并且配置静态路由使得他们能两两互通，并且在r4上搭建telnet服务，让r1远程登录。

一，各路由器的端口ip具体配置

一，r1端口配置

Sys（进入系统视图）

Sys r1(改名r1)

ip add 192.168.1.1 24（ip配置）

后面会省略，只写关键步骤

二.r2端口配置

三，r3端口配置

四,r4端口配置

现在 4个路由器的ip已经配置成功了 r1和r2能互通，r2能和r3能互通，r3能和r4互通（因为是直连的）要想两两互通，那么r1要和r3互通，r1要和r4互通，r2要和r4互通，那么就需要给r1和r2配置静态路由

二.配置静态路由

1.r1和r3配置静态路由

1.1：r1

ip route-static 192.168.2.0（目标网段） 24（编码） 192.168.1.2（下一跳）

1.2 r3配置到r1的静态路由（传输是双向的）

ip route-static 192.168.1.0 24 192.168.2.2

也可以看到现在 r3能ping r1 了。

2.r1和r4配置静态路由

2.1 r1-r4 静态路由

ip route-static 192.168.3.0 24 192.168.1.2

2.2 r4-r1 静态路由

ip route-static 192.168.1.0 24 192.168.3.3

现在r1和r4还不能ping通 因为r4和r2不能ping通 r4要经过r2才能到达r1

3.r2和r4配置静态路由

3.1 r2-r4 静态路由

ip route-static 192.168.3.0 24 192.168.2.3

3.2 r4-r2 静态路由

ip route-static 192.168.2.0 24 192.168.3.3

现在r4 ping r1 r2 展示

OK，现在能两两互通了。

开始在r4搭建telnet服务。

R1登录r4
